# 渐进式交付计划（Skill 迭代路线）

## 阶段 M0：骨架可运行

目标：页面可打开、主框架可见

交付：
- 基本布局（头部、输入、画布、代码、控制）
- 单解法静态分镜数据

通过条件：
- 本地可运行
- 基础交互可点击

## 阶段 M1：核心交互闭环

目标：用户可完整走通一次算法演示

交付：
- 输入校验 + 随机合法数据
- 播放控制与快捷键
- 代码高亮与步骤同步

通过条件：
- 单解法全流程可用
- 快捷键与进度条可用

## 阶段 M2：高信息密度可视化

目标：显著提升教学效果

交付：
- 数据流箭头与标签系统
- 空节点、状态迁移、结构联动
- 画布布局避让优化

通过条件：
- 新用户可在 3 秒内理解当前动作意图

## 阶段 M3：多解法对比

目标：同题多解法独立演示、可切换对比

交付：
- 独立解法容器
- 独立步骤与代码映射
- 切换时状态正确重置

通过条件：
- 多解法切换后无串状态问题

## 阶段 M4：发布与质量封版

目标：达到可对外演示质量

交付：
- CI 绿灯
- Pages 发布
- 验收报告与分数

通过条件：
- Rubric >= 80
- 无 P0/P1

## 使用建议

- 新题目至少做到 M2
- 热门题或教学演示题建议做到 M4
