## 3. 必过集合（Blocking E2E）

以下 E2E 用例必须 100% 通过：

## 4. 执行阈值（明确）

- 执行环境：Chromium，视口 1920x1080。
- 核心链路允许重试 0 次（默认不重试）；仅 flaky 场景允许最多 2 次重跑并留痕。
- 最终结论规则：
  - 必过集合任一失败 => AC-017 = FAIL => 总结论 FAIL
  - 必过集合全通过，且 AC-001~AC-018 都 PASS => 总结论 PASS
