# 网站需求目录（Skill 需求基线）

## A. 页面结构

1. 单屏布局：输入、画布、代码、控制面板首屏可见
2. 顶部标题：LeetCode 题号 + 中文标题，居中展示
3. 标题链接：新标签页打开题目页面
4. 左上返回：新标签页打开 Hot 100
5. 右上操作：算法思路按钮 + GitHub 徽标 + Star

## B. 输入系统

1. 单行紧凑输入条
2. 自定义输入 + 样例按钮 + 随机合法数据
3. 非法输入即时提示，不触发演示更新
4. 合法输入自动触发分镜重置和重算

## C. 多解法系统

1. 每种解法独立步骤序列
2. 每种解法独立代码映射
3. 每种解法独立画布渲染参数与镜头划分
4. 切换解法时重置当前播放状态

## D. 代码面板

1. 支持 Java/Python/Go/JavaScript
2. 默认 Java，语言偏好持久化
3. 行号、语法高亮、当前行高亮
4. 变量值旁注（watch 区 + 行联动）
5. 避免双向滚动；必要时优先保证可读性

## E. 画布系统

1. 支持拖拽平移与缩放
2. 默认居中，避免元素堆叠
3. 节点/边/数组/哈希/栈队列可视化
4. 状态迁移必须有箭头和文字说明
5. 空节点必须显式绘制（null）
6. 数据流（赋值、比较、指针移动）必须可视化

## F. 控制系统

1. 上一步 [←]
2. 下一步 [→]
3. 播放/暂停 [Space]
4. 重置 [R]
5. 速度切换（自定义组件）
6. 可拖拽进度条（绿色已播、灰色未播）

## G. 社群与品牌

1. 右下角交流群悬浮球
2. hover 显示二维码，比例不变形
3. 提示文案：扫码发送 leetcode 加入

## H. 工程与发布

1. 端口使用 30000-65535（避免默认冲突）
2. CI 必须执行 lint/type-check/build
3. GitHub Actions 发布 Pages
4. README 极简且准确
