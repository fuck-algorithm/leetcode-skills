# 测试与验收标准（Skill 基线）

## 1. 测试分层

### 1.1 单元测试层（新增）

- 框架：Vitest + jsdom + @testing-library/react
- 目录：`tests/unit/`
- 目标：验证 hooks、输入校验、数据转换等纯逻辑
- 覆盖率门槛：lines >= 60%, functions >= 60%, branches >= 50%
- 规则：新增 `features/` 或 `services/` 模块必须配套单元测试

### 1.2 逻辑层（集成）

重点覆盖：
- 输入校验器（合法/非法边界）
- 随机数据生成器（约束符合）
- 分镜绑定（anchorId -> code lines）
- 缓存策略（TTL、fallback）

### 1.3 组件层（集成）

重点覆盖：
- 代码面板语言切换和持久化
- 控制面板快捷键和进度条拖拽
- 解法切换后重置逻辑

### 1.4 页面层（E2E）

重点覆盖：
- 页面首屏关键元素可见
- 输入、播放、拖拽、切换语言、切换解法全链路
- 外链行为（新标签）

## 2. 测试准入门槛（Entry Criteria）

进入验收测试前必须满足：

1. 关键需求功能已开发完成
2. 本地 dev 服务可访问
3. lint/type-check/build 通过
4. 至少 1 套可复现输入样例

## 3. 测试退出门槛（Exit Criteria）

通过验收必须满足：

1. P0/P1 缺陷为 0
2. 核心 e2e 用例通过率 100%
3. 验收清单 AC-001~AC-018 全部通过或有明确豁免
4. 最终判定模型 B1-B7 全部通过

## 4. 缺陷分级

- P0：页面不可用、崩溃、关键链路不可操作
- P1：核心功能存在明显错误（分镜错位、拖拽失效）
- P2：非核心体验问题（样式对齐、轻微视觉不一致）
- P3：优化建议

## 5. 验收证据要求

每次验收结果必须带证据：

1. 命令执行输出（lint/type-check/build/e2e）
2. 关键页面截图（首屏、播放中、切语言、切解法）
3. 缺陷记录与修复说明
4. 验收轮次记录（Round N）及本轮结论（PASS/FAIL）

## 6. 强制回归触发条件

以下变更触发全量回归：

1. 画布渲染器改动
2. 分镜绑定机制改动
3. 控制面板交互改动
4. 缓存策略改动

## 7. 失败闭环循环（强制）

当任一 Blocking Gate 失败时，必须进入闭环循环：

1. 记录失败证据：命令输出、trace、截图、复现输入
2. 失败分级：P0/P1/P2/P3
3. 根因定位：实现缺陷 / 测试脚本缺陷 / 环境问题
4. 修复并回归：先跑受影响集，再跑核心冒烟必过集合
5. 更新验收报告：记录本轮失败项、修复动作、复测结果
6. 重新执行最终判定模型（B1-B7）

补充规则：

- 同一 flaky 用例最多重试 2 次；仍失败按真实缺陷处理
- 禁止跳过失败 Gate 直接结束任务
- 循环终止条件只有一个：最终结论为 PASS
