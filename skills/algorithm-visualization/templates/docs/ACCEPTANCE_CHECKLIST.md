# 验收清单（AC-ID，逐条可执行）

> 验收必须在 `1920x1080` 视口执行，除非条目内另有说明。
> 每条都要填 `PASS/FAIL/N/A`，并附证据。

| AC-ID | 验收项 | 验收操作（必须执行） | 通过标准（量化） | FAIL 判定 |
| --- | --- | --- | --- | --- |
| AC-001 | 单屏布局成立 | 打开首页后不滚动，检查标题/输入/画布/代码/控制是否可见 | `document.documentElement.scrollHeight <= window.innerHeight + 4` 且五大区域可见 | 出现纵向滚动或任一区域不可见 |
| AC-002 | 标题与题目链接正确 | 点击标题 | 标题文本包含“题号+中文标题”；链接为 LeetCode 题目地址；`target="_blank"` | 标题文本错误、链接错误或非新标签 |
| AC-003 | Hot100 链接正确 | 点击左上 Hot100 | href 为 `https://fuck-algorithm.github.io/leetcode-hot-100/` 且新标签打开 | 链接不一致或未新标签 |
| AC-004 | GitHub 徽标链接正确 | 点击右上 GitHub 徽标 | href 为当前仓库 URL 且新标签打开 | 链接错误或未新标签 |
| AC-005 | Star 获取+缓存+失败回退 | 先在线加载一次，再模拟 API 失败刷新 | 有缓存时展示缓存值；无缓存且 API 失败时显示 `0`；缓存 TTL=3600s | 失败后显示空白/NaN/报错崩溃 |
| AC-006 | 多解法独立（如有） | 进入解法A播到中间，切到解法B再切回A | 切换时 step 重置到初始；A/B 步骤与高亮互不串状态 | 切换后沿用另一解法状态 |
| AC-007 | 输入栏完整可用 | 分别测试：非法输入、合法输入、样例、随机 | 非法输入有错误提示且不重算；合法输入触发重算并回到 Step 1；样例与随机都能产出合法数据 | 非法触发重算、合法不重算、样例/随机失效 |
| AC-008 | 代码四语言+记忆 | 切换 Java/Python/Go/JS，刷新页面 | 四语言按钮都可切；默认 Java；刷新后保留上次语言；存在行号与当前行高亮 | 缺语言、刷新丢失语言、无行号/无高亮 |

| AC-010 | 数据结构联动可视化 | 播放至少 3 步，观察栈/队列/哈希联动区 | 至少一个联动结构随步骤发生可见变化（增删改） | 步骤变化但联动区静态不变 |
| AC-011 | 快捷键+进度拖拽 | 依次按 `→` `←` `Space` `R`，再拖拽进度条到 80% | `→` 步进+1，`←` -1，`Space` 切换播停，`R` 回到初始；拖拽后 currentStep 跳转到对应位置 | 任一快捷键无效或拖拽不跳步 |
| AC-012 | 速度记忆 | 将速度切到非默认（如 2x），刷新页面 | 刷新后速度保持为上次值 | 刷新后恢复默认速度 |
| AC-013 | 交流群二维码比例 | hover 悬浮球显示二维码 | 二维码可见；渲染宽高比与图片原始宽高比偏差 < 2% | 不显示二维码或明显拉伸变形 |
| AC-014 | 无紫色主题 | 运行 `npm run qa:guard` 并人工检查首屏 | 脚本通过且首屏无紫色主视觉/强调色 | 脚本报紫色 token 或肉眼可见紫色主色 |
| AC-015 | Pages 部署配置有效 | 检查 `.github/workflows/deploy.yml` | 包含 `npm ci/lint/type-check/build/configure-pages/upload-pages-artifact/deploy-pages` 全步骤 | 缺任一步骤或 workflow 文件不存在 |

## 必过规则（Blocking）

- 所有 AC 默认都是 Blocking。任一 AC=FAIL，最终结论必须是 `FAIL`。
- 仅 AC-006 在“题目无多解法”时可填 `N/A`，并在验收报告写原因。
- 每条 AC 必须附证据（截图、日志、命令输出、trace 之一或组合）。
